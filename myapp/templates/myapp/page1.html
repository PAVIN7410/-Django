Используем функцию, которая больше подходит для отображения целых html-страниц. Django, как и Flask, требует хранить html-файлы в папке templates, но эту папку нужно создавать внутри main. В официальной документации Django сказано, что внутри папки templates должна быть ещё одна папка, которая называется в точности так же, как приложение, а уже внутри этой папки — html-файлы.




Django требует эту папку, потому что при разработке больших проектов приложений может быть много, и внутри каждого приложения будет создана папка templates. При работе Django будет как бы объединять все эти папки в одну. Тогда файлы с одинаковыми названиями будут путаться (index.html во всех приложениях, например). 

from django.shortcuts import render
from django.http import HttpResponse
def index(request):
    return render(request, 'main/index.html')
